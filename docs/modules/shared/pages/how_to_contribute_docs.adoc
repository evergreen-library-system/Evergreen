= How to Contribute Documentation =
:toc:

We want everyone to be able to contribute documentation to the Evergreen project. We've made it as easy as possible. Please follow these simple steps to get your computer ready for contributions!

Here is a video going over this process. https://youtu.be/0KcfFiqCtP4[AsciiDoc and Antora] : Presented by Lynn Floyd, Evergreen Indiana and Blake Henderson, MOBIUS at the Evergreen International Online Pre-Conference 2021. 

== Create a GitHub Account ==

NOTE: Screenshots of the Github website might be out of date. Their website might look different today.

. Open a web browser and go to: https://github.com/[https://github.com/]
. Create an account by clicking "Sign Up"
+
image::how_to_contribute_docs/github_signup.png[Github Signup Screenshot]
+
. Fill in your information
+
image::how_to_contribute_docs/github_create_account.png[Github Create Account Screenshot]

== Fork the Community Evergreen Code Repository ==

We will need to make a copy of the Evergreen documentation. This will allow you to make edits to your copy.

. Find the Evergreen repository (evergreen-library-system/Evergreen) https://github.com/evergreen-library-system/Evergreen[https://github.com/evergreen-library-system/Evergreen]

. Click on "Fork"
+
image::how_to_contribute_docs/github_click_fork.png[Github Click Fork Screenshot]
+
. This process will make a copy of the repository into your personal GitHub account.

== Install the GitHub Desktop Software ==

We will need to download and install the GitHub Desktop software onto your computer. This software will act as liaison between your computer and GitHub.

. Open the web page where you can download the free installer https://desktop.github.com/[https://desktop.github.com/]
+
image::how_to_contribute_docs/github_desktop_download_site.png[Github Desktop Download Screenshot]
+
. Save the file to your computer.
. Execute the installer file.

== Clone Your Evergreen Repository ==

Once the Github Desktop software is installed, it should automatically launch itself. If not, you will find the software on your desktop or Start Menu.

. Sign the Github Desktop software into your GitHub Account
.. Click "File" -> "Options"
+
image::how_to_contribute_docs/github_desktop_click_option.png[Github Desktop Click Options Screenshot]
+
.. Click on "Accounts" -> "Sign In"
+
image::how_to_contribute_docs/github_desktop_click_signin.png[Github Desktop Click Sign in Screenshot]
+
.. Click on "Continue in Browser"
+
image::how_to_contribute_docs/github_desktop_singin_with_browser.png[Github Desktop Sign in with Browser Screenshot]
+
.. Sign into your account (if you were already signed in, then you can ignore this instruction)
.. Once signed in, you will be prompted by your browser to launch the GitHub Desktop Application again
+
image::how_to_contribute_docs/github_desktop_singin_with_browser_launch_app.png[Github Desktop Browser launch App Screenshot]
+
. Now that the GitHub Desktop software has been authenticated with your account, you should notice that you have your Forked Evergreen repository listed as an option. 
+
image::how_to_contribute_docs/github_desktop_clone_from_internet.png[Github Desktop Clone Evergreen Screenshot]
+
. Click on the Evergreen repository and then click the button at the bottom "Clone XXXX/Evergreen"
+
image::how_to_contribute_docs/github_desktop_clone_evergreen.png[Github Desktop Clone Evergreen Screenshot]
+
. Once cloned, you will be asked this question (pictured below). Choose "To contribute to the parent project" and click "Continue"
+
image::how_to_contribute_docs/github_desktop_planning_on_fork.png[Github Desktop Planning Fork Screenshot]

NOTE: From now on, you won't need to perform the previous steps. You can skip to "Make a Branch" to make new changes in the future.

== Make a Branch ==

A Git repository has a concept of a "branch". Which means you are making changes that do not affect the rest of the community. You are making changes to the documentation in your own private branch. You can invite others to work on your branch as well.

. From the GitHub Desktop Software, click on "Branch" -> "New branch"
+
image::how_to_contribute_docs/github_desktop_click_new_branch.png[Github Desktop Click New Branch Screenshot]
+
. Supply a name for your branch. It's a good idea to name it with a good summary of what your changes will contain.
+
image::how_to_contribute_docs/github_desktop_make_branch.png[Github Desktop Make Branch Dialog Screenshot]
+
. Confirm that you are working in your branch
+
image::how_to_contribute_docs/github_desktop_quick_tour.png[Github Desktop Quick Tour Screenshot]

== Make Your Edits ==

The Evergreen documentation is written in AsciiDoc. AsciiDoc is plain old text with some basic formatting stuff around it. AsciiDoc is great because it takes away so much of the pain of formatting the words and allows you to focus more on the content rather than the formatting. Looking at AsciiDoc text can be a bit confusing at first but please stick with it. It will make more sense in no time!

=== A quick word on images ===

Each section of the Evergreen documentation is divided into "modules". Each module has a special place where we save our image files. Images are then referred to in the documentation.

For example:

This document is here:

[source,bash]
----
Evergreen/docs/modules/shared/pages/how_to_contribute_docs.adoc
----

And the images are here:

[source,bash]
----
Evergreen/docs/modules/shared/assets/images/how_to_contribute_docs/
----

When you are editing your document, you will refer to your image like this:

[source]
----
image::how_to_contribute_docs/filename.png
----

== Commit Your Changes ==

Now that you've completed your changes on your computer, you need to commit them to the repository and push those changes to the cloud. This is done using the GitHub Desktop software.

. Open the GitHub Desktop Software
+
image::how_to_contribute_docs/github_desktop_commit_changes.png[Github Desktop Commit Changes Screenshot]
+
.. Be sure and double check that you are on your branch
.. GitHub Desktop will display a summary of your changes on the left pane
.. If you click on one of the files that you've edited, GitHub Desktop will display the specific changes to that file in the right pane

=== Supply a Commit Message ===

Your commit message needs to be supplied in the highlighted box (bottom left)

When you are satisfied with your commit message (and description), click on "Commit to <branch name>"

=== Commit to the GitHub Cloud ===

Now that you have committed a change to your copy of the repository, you will no longer see any changes in the "Changes" section. You can see your commit in the "History" tab. In order for everyone in the Evergreen community to see your changes, you will need to publish it to  the internet.

Click on "Publish branch".

image::how_to_contribute_docs/github_desktop_publish.png[Github Desktop Publish Branch Screenshot]

=== Make a "Pull Request" ===

This is a way to flag down the Evergreen community core committers and let them know that this change is pending approval.

. From the GitHub Desktop software, you should be prompted to "Create a Pull Request"
+
image::how_to_contribute_docs/github_desktop_create_pull_request.png[Github Desktop Create a Pull Request Screenshot]
+
. This will open your default web browser and prompt you to confirm some of the details
+
image::how_to_contribute_docs/github_web_create_pull_request.png[Github Web Create a Pull Request Screenshot]
+
. It's nice to see "Able to merge"
. Confirm that this pull request is compared with *master* on the left and with *your branch* on the right
. Click on "Create pull request"
. Github will run a few checks, to make sure that your changes match our style guide
  and won't cause problems when they are published on docs.evergreen-ils.org.  These checks will take a few minutes.
    * If everything is good, Github will add a green
  checkmark icon next to your pull request.
+
image::how_to_contribute_docs/checkmark_pr.png[A github pull request shows a green checkmark icon immediately after the title of the pull request]
+
    * If there is an issue, Github will add a red X icon.  You can click the red X
    icon for more details.  To address the issue, you can adjust your version of
    the docs locally, commit your changes, and publish your branch again.

== Follow Up on Launchpad ==

The Evergreen Community bug tracking platform is called Launchpad. You can see the main page here: https://launchpad.net/evergreen[https://launchpad.net/evergreen]. It's common Evergreen workflow to track and discuss any needed Evergreen changes on Launchpad. If your documentation change has an associated Launchpad bug, be sure and update the bug with details about your contribution on GitHub! This gives more ways for the community to find your contribution.

== Contributing Documentation via GitHub Browser ==

You can also use GitHub in your browser instead of the desktop application. Follow the previous steps to set up your account, add the Evergreen repository, and create your fork. Then, follow the instructions below to contribute text and images.

=== Contributing Text Changes ===

. Log in to your GitHub account and make sure you’re on the main Evergreen repository. If you are not, click the hamburger menu at the top left corner of the page and select the Evergreen repository.
+
image::how_to_contribute_docs/main_repository.png[Evergreen main repository path]
+
. Navigate to the page where you want to make changes. You can use the search bar above the modules (“Go to file”) to find the file based off of the URL of the page in the documentation. The file name will be the same as the end of the URL but with _adoc_ instead of _html_. For example, to find the file for /admin/circulation_limit_groups.html, you can search “/admin/circulation_limit_groups.adoc” or “circulation_limit_groups.adoc”
+
image::how_to_contribute_docs/search_bar.png[GitHub file search bar]
+
. Or you can navigate through the menu manually, starting with docs > modules. Using the same example, the path to access the page would be docs > modules > admin > pages > circulation_limit_groups.adoc.
. Once on the page, click the pencil icon to edit.
+
image::how_to_contribute_docs/edit.png[""]
+
. Make changes within the text, following AsciiDoc formatting rules. During or after you’ve made your changes, you can click on the *Preview* tab to preview the updated document.
. When all text changes are complete, click *Commit Changes* (or, if you want to cancel, click Cancel Changes).
+
image::how_to_contribute_docs/commit_changes.png[""]
+
.  A pop-up will appear. Add a commit message title and description. Generally, documentation changes start with the “Docs:” tag to differentiate them from code changes; for example, “Docs: Updated terminology in circulation_limit_groups.adoc”. If there is an associated bug in Launchpad, it is helpful to include the bug number.
+
image::how_to_contribute_docs/commit_message_title.png[Commit message pop-up form]
+
. Click *Propose changes*.
+
image::how_to_contribute_docs/propose_changes.png[Form with commit title, message, and propose changes button]
+
. You will be led to a *Comparing changes* screen, where you can see the original text lines in red, and the edited lines in green.
+
[NOTE]
======
If you are planning to submit corresponding screenshots or additional text changes to other pages, stop here and go to the next section.
======
+
. Click *Create pull request*.
+
image::how_to_contribute_docs/comparing_changes_create.png[View of comparing changes screen with edits highlighted]
+
. Click *Create pull request* again.


Checks will run to ensure there are no issues with the pull request. 

image::how_to_contribute_docs/doc_checks.png[Example checks pending]


If the checks pass, the pull request has been submitted successfully.

image::how_to_contribute_docs/doc_checks_passed.png[Example checks passed]


Once your pull request has been submitted, you can view it under the main Evergreen repository *Pull requests* tab.

image::how_to_contribute_docs/open_pull_requests.png[View of comparing changes screen with edits highlighted]

If the change is related to a bug in Launchpad, copy the URL of the pull request and paste it as a comment in the bug report. See xref:shared/how_to_contribute_docs.adoc#_follow_up_on_launchpad[Follow up on Launchpad] for additional information.

==== Including Additional Changes in Same Pull Request ==== 

If you would like to add updated screenshots to the same page, or make additional text changes to other pages, within the same pull request:

. Click **Code** at the top left of the screen.
+
image::how_to_contribute_docs/return_to_code.png[Link back to main code page]
+
. Click on the link of the patch.
+
image::how_to_contribute_docs/patch_link.png[Patch name link]
+
. Complete steps 2 through 8 above.
. Make sure *Commit directly to the patch-27 branch* is selected.
+
image::how_to_contribute_docs/commit_directly.png[""]
+
. Click *Commit changes*.
. Click *Compare & Pull request* in the yellow bar at the top of the page.
+
image::how_to_contribute_docs/compare_and_pull_request.png[""]
+
. Update the title if needed (the other pull requests listed below in the Commits list).
. Click *Create pull request*.

Checks will run to ensure there are no issues with the pull request. If the checks pass, the pull request has been submitted successfully.

=== Contributing Image Changes ===

Users may not have the permissions to add or update image files directly to the main Evergreen repository. To add screenshots, you will need to do so from your Evergreen fork (Git username/Evergreen). 

When adding images, first make sure the image tag in the documentation is correct. If the image tag needs to be updated, follow the steps above to update the text and then add your images within the same pull request.

Once in your fork, make sure the code is synced and then follow the steps below:

. Find the corresponding image folder to add the image. Follow the same path as finding the text file but click on the _assets/images_ subfolder (as opposed to the _pages_ folder) to find the corresponding subfolder where the images are stored.
. To upload the image, click *Add file*, *Upload files*, and drag and drop your image into the folder.
. Leave the commit directly to the patch option selected. Then, click *Commit changes*.
. Click *Compare & Pull request* in the yellow bar at the top of the page.
. On the comparing changes screen, make any final updates to the title and description if needed.
. Click *Create pull request*.
